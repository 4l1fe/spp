Part 5: Time Series Prediction with Prophet in Python

Reading Time: 4 minutes

Welcome to the fifth part Time Series Prediction with Prophet in Python of the Comprehensive Guide on Time Series Analytics, Decomposition and Prediction with Python.

In this part we introduce the Prophet forecasting model open sourced by Facebook and conduct out-of-sample predictions on our time series using the Python programming language. This modules requires minimal effort to perform a prediction, however, it requires a specific data input format in order to predict efficiently. In this article we will learn how.

The code for all calculations and graphs can be found on my GitHub.

The whole series will use the international airline passenger data set of Box and Jenkins (1973). It can be found freely on the web or in this GitHub repository.

Time Series Forecasting with Prophet

Prophet is a procedure for forecasting time series data. It is based on an additive model where non-linear trends are fitted with yearly and weekly seasonality, plus holidays. It works best with daily periodicity data with at least one year of historical data. Prophet is robust to missing data, shifts in the trend, and large outliers. – Prophet

The Model

Forecasting in the domain of Prophet is a curve-fitting task. The underlying model has an additive form

where denotes a trend function modeling non-periodic changes, denotes seasonality modeling periodic changes and representing the effects of holidays. This model assumes time as its only regressor, however, linear and non-linear transformations are included if it increases the models fit.

Preparing Data input

The input to Prophet is always a pandas.DataFrame object with two columns and headers: ds and y. The ds (datestamp) column must contain a date or datetime object (either is fine). The y column must be numeric, and represents the time series to forecast.

Model Fitting

The Prophet syntax is very minimalistic. We first instantiate the Prophet class. Then we fit the model to our training data. We then define the time horizon over which the model should perform a forecast starting from the last historic data point.

1
2
3
4
5
6

8
# instantiate the Prophet class
mdl = Prophet(interval_width=0.95)
 
# fit the model on the training data
mdl.fit(df_train)
 
# define future time frame
future = mdl.make_future_dataframe(periods=24, freq='MS')

The predict method assigns each row in future a predicted value yhat.

1
2
3
# generate the forecast
forecast = mdl.predict(future)
forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']].tail()

Evaluating the Model Fit

Since we have log transformed our time series in the beginning we now need to revert this transformation by applying the exponential function. We then compute the Mean Squared Error (MSE) and Root Mean Squared Error (RMSE) to compare the model performance with the previously applied ARIMA model.

1
2
3
4
5
6
7
# retransform using e
y_hat = np.exp(forecast['yhat'][120:])
y_true = np.exp(df_test['y'])
 
# compute the mean square error
mse = ((y_hat - y_true) ** 2).mean()
print('Prediction quality: {:.2f} MSE ({:.2f} RMSE)'.format(mse, math.sqrt(mse)))

Summary

In this article we learned how to effectively apply the Prophet framework for time series prediction. The modules syntax is is minimal and the computation time and model accuracy appear better then the results obtained from the ARIMA prediction models applied previously.
